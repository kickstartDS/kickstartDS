# Sections

## Style

A section style is mainly expressed by the given background color. <br/>
You should make sure that the chosen background color always offers enough contrast to display various font colors!

## Frame Width

The `width` propertiy determines the with of the frame the content will be displayed in.

| Setting   | Token                                | Application                                                               |
| --------- | ------------------------------------ | ------------------------------------------------------------------------- |
| `default` | `--l-section--content-width-default` | The default frame width                                                   |
| `narrow`  | `--l-section--content-width-narrow`  | A narrow frame width                                                      |
| `wide`    | `--l-section--content-width-wide`    | A wide frame width                                                        |
| `max`     | `--l-section--content-width-max`     | The content spans over the entiere screen                                 |
| `full`    | `--l-section--content-width-full`    | The content spans over the entiere screen,<br/> without `content-padding` |

#### Screen spanning layouts

For creating full screen layouts there are options of `max` and `full`.<br/>
They both state that the frame spans over the entire screen.<br/>
The `max` option still leaves `content-padding` to the left and right browser border.<br/>
With the `full` option the content will be placed directly against the browser border.

## Space Before / Space After

Determines how much **vertical** spacing a section has.

| Setting   | Preview                                                       | Token                      | Application                                        |
| --------- | ------------------------------------------------------------- | -------------------------- | -------------------------------------------------- |
| `default` | <div className="spacing-preview spacing-section"></div>       | `--l-section--space`       | The default section spacing ensuring an even rythm |
| `small`   | <div className="spacing-preview spacing-section-small"></div> | `--l-section--space-small` | A smaller space between Sections                   |
| `none`    |                                                               | ---                        | No space between Sections                          |

#### Combining spacings

By combining the given `spaceAfter`/`spaceBefore` options, it's possible to move sections closer together.<br/>
That way you can create a visual connection between the content in two sections.<br/>
You can even make them seem as a single element by removing all spacing between them.

## Mode

#### `default`

The children will be placed next to each other, stretched across the full section width.

<Section
  ks-theme="section-preview"
  spaceBefore="small"
  spaceAfter="small"
  width="full"
>
  <div className="section-preview--child">Content</div>
  <div className="section-preview--child">Content</div>
</Section>

#### `tile`

The children will be placed next to each other, in a pre set column width.

<Section
  ks-theme="section-preview"
  spaceBefore="small"
  spaceAfter="small"
  mode="tile"
  width="full"
>
  <div className="section-preview--child">Content</div>
  <div className="section-preview--child">Content</div>
</Section>

#### `list`

The children will be placed on top of each other, stretched across the full section width.

<Section
  ks-theme="section-preview"
  spaceBefore="small"
  spaceAfter="small"
  mode="list"
  width="full"
>
  <div className="section-preview--child">Content</div>
  <div className="section-preview--child">Content</div>
</Section>

## Customisation

Customisation is typically done via `scss` files.<br/>
The default token are declared on the `.l-section__container` selector and can be overritten there.<br/>
To increase the selector accuracy, its recommended to wrap them into a `:root` selector.

<CH.Code>

```scss _section-token.scss
:root,
[ks-inverted],
[ks-theme] {
  .l-section__container {
    //Adjust background styles
    --l-section--background: var(--ks-background-fg-alpha-1);
    --l-section--background-accent: var(--ks-background-primary-to-bg-8);
    --l-section--background-dark: var(--ks-background-primary-to-bg-2);

    //Adjust vertical spacing
    --l-section--space: var(--ks-spacing-xl);
    --l-section--space-small: var(--ks-spacing-m);

    //Adjust gutter spacing
    --l-section--gutter: var(--ks-spacing-m);
    --l-section--gutter-small: var(--ks-spacing-s);
    --l-section--gutter-large: var(--ks-spacing-l);

    //Adjust frame widths
    --l-section--content-width-wide: 87.5rem;
    --l-section--content-width-default: 77.5rem;
    --l-section--content-width-narrow: 62.5rem;

    //Adjust grid settings
    --l-section_col--min-width: 5rem;
  }
}
```

```scss _section.scss
@use "./section-tokens";
```

</CH.Code>

KickstartDS Sections use `CSS Grid`.<br/>
The child elements will be placed in columns, which are arranged in rows.<br/>
To get deeper into the workings of `CSS Grid` we recommend this article:<br/>
https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_Grid_Layout

The minimum width of a column is determined by `--l-section_col--min-width`.

The maximum width of a column is determined by `--l-section_col--max-width`.<br/>
The default value is `1fr`. (Recommended for responsive layouts)

The amount of columns fit in one row is determined by `--l-section_col--repeat`.<br/>
The default value is `auto-fit`. (Recommended for responsive layouts)

### Fill Space

If you want the available space to be efficiently used, it is recommended to leave the `max-width` token set to its default value of `1fr`.

<div className="col-fill">
  <Section
    ks-theme="section-preview"
    spaceBefore="small"
    spaceAfter="small"
    width="full"
  >
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>
  </Section>
</div>

### Fixed Width

If you dont want your elements to fill the available space, you can set a `max-width` value in addition to `min-width`.

`--l-section_col--min-width: 5rem`<br />
`--l-section_col--max-width: 10rem`

### Scaling Width

By using a percentage on the `min-width` token, you can create a scaling layout.<br/>
It will keep its proportions, no matter the screen size.

<div className="col-scale">
  <Section
    ks-theme="section-preview"
    spaceBefore="small"
    spaceAfter="small"
    width="full"
  >
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>

  </Section>
</div>
<br/>

`--l-section_col--min-width: 40%`<br/>

### Set Columns Amount

If you use a custom value, make sure the columns do not exceed the avialable space in your layout (especially on narrow screens).<br/>
You can do so by balancing out the `min-width` in correspondence to the set column amount.
