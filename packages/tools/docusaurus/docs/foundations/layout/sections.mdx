## Frame Width

The `width` propertiy determines the with of the frame the content will be displayed in.

| Setting   | Token                                | Application                                                                   |
| --------- | ------------------------------------ | ----------------------------------------------------------------------------- |
| `default` | `--l-section--content-width-default` | The default frame width of content                                            |
| `narrow`  | `--l-section--content-width-narrow`  | A narrow frame width of content                                               |
| `wide`    | `--l-section--content-width-wide`    | A wide frame width of content                                                 |
| `max`     | `--l-section--content-width-max`     | The content will span over the entiere screen                                 |
| `full`    | `--l-section--content-width-full`    | The content will span over the entiere screen,<br/> without `content-padding` |

## Space Before / Space After

Determines how much **vertical** spacing a section has

| Setting   | Preview                                                       | Token                      | Application                                        |
| --------- | ------------------------------------------------------------- | -------------------------- | -------------------------------------------------- |
| `default` | <div className="spacing-preview spacing-section"></div>       | `--l-section--space`       | The default section spacing ensuring an even rythm |
| `small`   | <div className="spacing-preview spacing-section-small"></div> | `--l-section--space-small` | A smaller space between Sections                   |
| `none`    |                                                               | ---                        | No space between Sections                          |

## Mode

#### `default`

The children will be placed next to each other, stretched across the full section width.

<Section
  ks-theme="section-preview"
  spaceBefore="small"
  spaceAfter="small"
  width="full"
>
  <div className="section-preview--child">Content</div>
  <div className="section-preview--child">Content</div>
</Section>

#### `tile`

The children will be placed next to each other, in a pre set column width.

<Section
  ks-theme="section-preview"
  spaceBefore="small"
  spaceAfter="small"
  mode="tile"
  width="full"
>
  <div className="section-preview--child">Content</div>
  <div className="section-preview--child">Content</div>
</Section>

#### `list`

The children will be placed on top of each other, stretched across the full section width.

## Grid settings

KickstartDS Sections use `CSS Grid`.<br/>
The child elements will be placed in columns, which are arranged in rows.

The minimum width of a column is determined by `--l-section_col--min-width`.

The maximum width of a column is determined by `--l-section_col--max-width`.<br/>
The default value is `1fr`. (Recommended for responsive layouts)

The amount of columns fit in one row is determined by `--l-section_col--repeat`.<br/>
The default value is `auto-fit`. (Recommended for responsive layouts)

### Fill Space

If you want the available space to be efficiently used, it is recommended to leave the `max-width` token set to its default value of `1fr`.

<div className="col-fill">
  <Section
    ks-theme="section-preview"
    spaceBefore="small"
    spaceAfter="small"
    width="full"
  >
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>
  </Section>
</div>
<br />

`--l-section_col--min-width: 5rem`<br/>

### Fixed Width

If you dont want your elements to fill the available space, you can set a `max-width` value in addition to `min-width`.

<div className="col-fixed">
  <Section
    ks-theme="section-preview"
    spaceBefore="small"
    spaceAfter="small"
    width="full"
  >
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>
  </Section>
</div>
<br />

`--l-section_col--min-width: 5rem`<br />
`--l-section_col--max-width: 10rem`

### Scaling Width

By using a percentage on the `min-width` token, you can create a scaling layout.<br/>
It will keep its proportions, no matter the screen size.

<div className="col-scale">
  <Section
    ks-theme="section-preview"
    spaceBefore="small"
    spaceAfter="small"
    width="full"
  >
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>

  </Section>
</div>
<br/>

`--l-section_col--min-width: 40%`<br/>

### Set Columns Amount

If you use a custom value, make sure the columns do not exceed the avialable space in your layout (especially on narrow screens).<br/>
You can do so by balancing out the `min-width` in correspondence to the set column amount.

<div className="col-limit">
  <Section
    ks-theme="section-preview"
    spaceBefore="small"
    spaceAfter="small"
    width="full"
  >
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>
    <div className="section-preview--child">Content</div>

  </Section>
</div>
<br/>

`--l-section_col--repeat: 4` <br/>
`--l-section_col--min-width: 5rem`
